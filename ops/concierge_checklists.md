# Concierge Checklists - Maroa Pilot

---

## 1. Daily Opening Checklist

**When:** Start of each working day (8:00 AM)

| # | Task | DoD |
|---|------|-----|
| 1 | Check WhatsApp for overnight messages | All new messages read |
| 2 | Prioritize by SLA urgency | Urgent items flagged |
| 3 | Review today's scheduled services | Know who, where, what time |
| 4 | Confirm technician availability | Tech confirms via WhatsApp |
| 5 | Send T-24h reminders (if applicable) | Reminders sent to tomorrow's clients |
| 6 | Update Leads sheet with new inquiries | All new leads logged |

**DoD for checklist:** All 6 items marked complete in daily log

---

## 2. New Lead Intake Checklist

**When:** New message from potential client

| # | Task | DoD |
|---|------|-----|
| 1 | Respond within SLA (<24h) | First response sent |
| 2 | Ask triage questions (name, address, # units) | Client provides info |
| 3 | Verify address is in pilot zone | Zone confirmed |
| 4 | Log in Leads sheet | Row created with all fields |
| 5 | Classify lead (Qualified / Not a fit / Waitlist) | Status updated |
| 6 | If qualified, move to scheduling | Scheduling started |

**DoD for checklist:** Lead has status + next action + owner in Sheet

---

## 3. Scheduling Checklist

**When:** Lead qualifies for service

| # | Task | DoD |
|---|------|-----|
| 1 | Check technician availability | Open slots identified |
| 2 | Propose 2-3 date/time options to client | Message sent |
| 3 | Client confirms slot | Confirmation received |
| 4 | Assign technician | Technician notified and confirmed |
| 5 | Create row in Services sheet | Service logged with all details |
| 6 | Send confirmation message with details | Client has: date, time, address, what to expect |

**DoD for checklist:** Service status = "Agendado", client + tech both confirmed

---

## 4. Pre-Service (T-24h) Checklist

**When:** Day before scheduled service

| # | Task | DoD |
|---|------|-----|
| 1 | Send reminder to client | Message delivered |
| 2 | Ask client to confirm availability | Client confirms |
| 3 | Send reminder to technician | Message delivered |
| 4 | Confirm technician has address + details | Tech confirms |
| 5 | Update service status | Status = "Confirmado" |

**DoD for checklist:** Both parties confirmed, status updated

---

## 5. Day-of-Service (T-2h) Checklist

**When:** 2 hours before service

| # | Task | DoD |
|---|------|-----|
| 1 | Send "on the way soon" message to client | Client notified |
| 2 | Confirm tech is on track | Tech confirms start time |
| 3 | Be available for real-time communication | WhatsApp monitored |

**DoD for checklist:** Client and tech ready, concierge on standby

---

## 6. During Service Monitoring

**When:** Service in progress

| # | Task | DoD |
|---|------|-----|
| 1 | Receive "En ruta" from tech | Logged with timestamp |
| 2 | Receive "Llegue" from tech | Logged with timestamp |
| 3 | Handle any client/tech issues immediately | Issues documented |
| 4 | Receive "Completado" from tech | Status updated |
| 5 | Receive photos (before/after) | Photos saved to folder |

**DoD for checklist:** Service status = "Completado", photos archived

---

## 7. Post-Service Closing Checklist

**When:** Service marked complete

| # | Task | DoD |
|---|------|-----|
| 1 | Thank client for their service | Thank you message sent |
| 2 | Send payment instructions/link | Payment details shared |
| 3 | Confirm payment received | Payment logged in Sheet |
| 4 | Update service status to "Pagado" | Status = "Pagado" |
| 5 | Archive service photos | Photos in correct folder |
| 6 | Log any notes/feedback | Notes in Services sheet |

**DoD for checklist:** Service row complete, payment confirmed, closed

---

## 8. Follow-Up (T+24-48h) Checklist

**When:** 1-2 days after service

| # | Task | DoD |
|---|------|-----|
| 1 | Send satisfaction check message | Message sent |
| 2 | Log client feedback | Feedback in Sheet |
| 3 | Send referral request with script | Referral ask sent |
| 4 | Log any referral leads | New leads captured |
| 5 | Flag any issues for review | Issues in Incidencias sheet |

**DoD for checklist:** Feedback logged, referral requested

---

## 9. Weekly Closing Checklist

**When:** Friday afternoon / Saturday after close

| # | Task | DoD |
|---|------|-----|
| 1 | Review all open leads - update status | No stale leads |
| 2 | Check all completed services are closed | All complete + paid |
| 3 | Count SLA compliance metrics | Numbers ready for review |
| 4 | Document any incidents | Incidencias sheet updated |
| 5 | Prepare summary for weekly review | Bullet points ready |

**DoD for checklist:** Ready for weekly review meeting

---

## 10. Incident Handling Checklist

**When:** Anything unexpected occurs

| # | Task | DoD |
|---|------|-----|
| 1 | Acknowledge issue to affected party | Response sent |
| 2 | Assess severity (minor/major/critical) | Severity logged |
| 3 | Take immediate corrective action | Action taken |
| 4 | Log in Incidencias sheet | Full details recorded |
| 5 | Notify Operations Lead if major/critical | Escalation complete |
| 6 | Follow up with affected party on resolution | Resolution confirmed |

**DoD for checklist:** Incident documented, resolved or escalated
