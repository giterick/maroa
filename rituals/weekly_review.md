# Weekly Review Ritual - Maroa Pilot

---

## Purpose

Review pilot performance, identify issues, make data-driven decisions, and align on next week's priorities.

---

## Meeting Details

| Attribute | Value |
|-----------|-------|
| **Frequency** | Weekly |
| **Day/Time** | Friday 4:00 PM (or Saturday morning) |
| **Duration** | 45-60 minutes max |
| **Attendees** | Marcia (Ops), Erick (Systems), Gabby (Backup - opcional), Tech Lead (optional) |
| **Format** | Video call or in-person |

---

## Pre-Meeting Prep (DoD: Ready by Thursday EOD)

### Owner: Marcia
- [ ] Update all LEADS statuses
- [ ] Close all completed SERVICES
- [ ] Log any missing INCIDENCIAS
- [ ] Calculate SLA compliance %
- [ ] Calculate FTFR %
- [ ] Note top 3 operational frictions

### Owner: Erick
- [ ] Update WEEKLY_METRICS sheet
- [ ] Calculate conversion rate
- [ ] Calculate gross margin per service
- [ ] Prepare revenue summary
- [ ] Note any system/data issues

---

## Red Flags That Trigger Extra Review

| Trigger | Action |
|---------|--------|
| Any KPI in "Stop" zone | Emergency review within 24h |
| Critical incident | Same-day debrief |
| 2+ weeks in "Pivot" zone | Decision meeting: continue/pause |
| Zero services in a week | Root cause analysis |

---

## Definition of Done for Weekly Review

- [ ] All metrics calculated and reviewed
- [ ] Incidents discussed and logged
- [ ] Frictions identified with proposed fixes
- [ ] Decisions documented
- [ ] Actions assigned with owners and dates
- [ ] Notes sent to team

---

## Retrospective Questions (Monthly or End of Pilot)

1. What worked well that we should keep?
2. What didn't work that we should stop?
3. What should we start doing?
4. What surprised us?
5. What would we do differently if starting over?
